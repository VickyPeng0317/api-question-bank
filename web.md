功能需求詳細說明

請根據以下 4 點更新系統邏輯與介面：

1. 難度分級 (Level Distinction)

• 資料面：所有考卷必須歸類為 初級 或 中級。

• 儀表板：

    ◦ 首頁需增加「難度切換」標籤 (Tabs) 或下拉選單，讓使用者選擇查看「初級」或「中級」的統計數據。

    ◦ 統計數據 (答對率、常錯 Tag) 需根據當前選擇的難度進行過濾。

2. 既有題庫載入 (Preset Loader)

• 初始化：網站載入時，檢查 LocalStorage 是否已有資料。若無，顯示「預設題庫列表」。

• 載入介面：在「題庫管理」區塊，列出 presetExams 中的項目。

    ◦ 顯示每個預設題庫的名稱、分類、難度。

3. 考題匯入優化 (Smart Import)

• 提供規格範例

• 手動匯入的考題暫存在 localstorage，並且可以匯出

4. 考試模式更新

• 篩選邏輯：進入考試設定頁面時，使用者需先選擇「難度 (初級/中級)」，接著選擇「科目分類 (Type)」。

• 隨機出題：

    ◦ 隨時考模式：僅會從「選定難度」且「選定科目」的題庫中隨機抽取 10 題。

    ◦ 模擬考模式：可選擇「全真模擬 (特定一份 JSON)」或「綜合測驗 (該難度+該科目下隨機 50 題)」。

介面設計重點

• 請使用 Tailwind CSS 設計一個乾淨的頂部導覽列，顯示當前所在的難度 (例如：左上角顯示 "AI 應用規劃師 - 初級題庫")。

• 儀表板使用卡片式設計區分不同科目的進度。

• 作答畫面：需顯示題目來源 (例如："114-4-AI-Subject2") 與 題號。

請提供完整的 index.html，包含所有 Vue 邏輯、CSS 樣式與範例資料。程式碼中請加入註解說明「如何修改預設題庫連結」。
